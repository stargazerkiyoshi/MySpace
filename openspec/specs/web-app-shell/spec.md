## MODIFIED Requirements

### 需求:占位路由页面
系统必须为核心页面提供路由入口，包括但不限于空间、节点、时间线、仪表盘等后续扩展入口；其中 Space 路由必须在已有最小 Space 业务闭环基础上继续演进，承载该 Space 下的 Node 列表、Node 创建入口、Node 详情查看与基础编辑能力，而不只是停留在静态占位内容。

#### 场景:切换到 Space 页面
- **当** 用户访问 Space 路由
- **那么** 系统必须展示 Space 列表或空状态，而不是仅显示静态占位内容
- **并且** 页面中必须存在创建新 Space 的最小交互入口

#### 场景:进入某个 Space 后查看 Node 区域
- **当** 用户从 Space 列表进入某个具体 Space
- **那么** Space 详情页必须展示该 Space 下的 Node 列表区域
- **并且** 页面中必须存在创建 Node、查看 Node 详情和编辑 Node 基础字段的最小交互入口
