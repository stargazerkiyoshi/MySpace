## 新增需求

### 需求:首页状态页必须承载当前与历史的联动入口
系统必须在首页状态页中承载当前状态与历史节点之间的最小联动入口，使用户能够从当前状态直接回到相关历史。

#### 场景:从首页查看当前与历史的关系
- **当** 用户进入首页状态页
- **那么** 页面必须展示当前状态与历史来源之间的最小关系信息
- **并且** 页面必须提供跳转到相关历史节点的真实入口

### 需求:Space 工作区必须支持结构视图切换入口
系统必须在 Space 工作区中提供进入 Node Graph View 的切换入口，使用户能够在列表、历史和结构可视化之间切换，而不离开当前 Space 页面。

#### 场景:在 Space 工作区切换到 Graph View
- **当** 用户位于某个 Space 的工作区页面
- **那么** 页面必须提供进入 Node Graph View 的切换入口
- **并且** 该入口必须与现有工作区视图保持一致的页面承载方式

### 需求:Graph View 必须承载图画布与信息面板的最小布局
系统必须在 Graph View 中承载图画布与信息面板的最小布局，以便用户在浏览结构时能够同时看到选中节点的上下文信息。

#### 场景:在 Graph View 中查看选中节点信息
- **当** 用户在 Graph View 中选中某个 Node
- **那么** 页面必须在同一工作区内展示与该节点相关的基础信息面板或详情摘要
- **并且** 页面不得强制跳转到独立详情路由才能完成这一步
