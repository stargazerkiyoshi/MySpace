## ADDED Requirements

### 需求:UI 文案支持中英文切换
系统必须允许用户在前端界面上切换 `zh-CN` 与 `en` 两种 UI 语言。

#### 场景:用户切换界面语言
- **当** 用户在前端界面中切换当前语言
- **那么** 系统必须立即用所选语言渲染当前页面中的 UI 文案

### 需求:当前语言必须被持久化
系统必须将用户当前选择的 UI 语言持久化到本地存储，以便页面刷新后保持一致。

#### 场景:刷新后保留语言选择
- **当** 用户已经切换到某种 UI 语言并刷新页面
- **那么** 系统必须继续使用上一次选择的语言渲染 UI 文案

### 需求:当前阶段只翻译前端 UI 文案
系统当前阶段必须只翻译前端 UI 文案，不得将数据库内容、用户输入或后端业务数据强制纳入翻译范围。

#### 场景:显示用户输入内容
- **当** 页面中展示来自用户输入或数据库存储的原始内容
- **那么** 系统必须保留其原始内容，而不是强制翻译该值

### 需求:当前已有关键界面必须接入双语
系统必须在应用壳层、主导航、Space 列表页、Space 详情页和公共提示中接入双语文案。

#### 场景:访问已覆盖页面
- **当** 用户访问当前已覆盖的壳层或 Space 相关页面
- **那么** 页面中的导航标题、说明文案、按钮文案、空状态与基础提示必须根据当前语言显示

## MODIFIED Requirements

### 需求:应用壳层与导航布局
系统必须提供稳定的应用布局层，用于承载主导航、页面容器和后续空间型协作页面入口；并且应用壳层必须提供语言切换入口，使用户能够在不离开当前页面的情况下切换 UI 语言。

#### 场景:在应用壳层切换语言
- **当** 用户位于前端任意已覆盖页面
- **那么** 用户必须能够通过应用壳层中的统一入口切换当前 UI 语言
- **并且** 切换后当前页面应立即按新语言更新文案
