## 新增需求

### 需求:当前状态必须具备可追溯的历史锚点语义
系统必须为首页当前状态提供稳定的历史挂接语义，使当前状态在历史结构中有明确落点，而不是只作为孤立摘要展示。

#### 场景:读取当前状态锚点
- **当** 前端请求某个 Space 的当前状态摘要
- **那么** 系统必须返回能够表达当前状态历史来源的最小结构
- **并且** 该结构必须至少包含当前主线锚点或当前状态来源节点中的一种明确语义

### 需求:首页必须支持从当前状态进入关键历史节点
系统必须允许用户从首页当前状态区域进入与当前状态相关的关键历史节点，而不是要求用户自行在时间线中重新定位。

#### 场景:查看当前状态来源节点
- **当** 首页状态区域展示某个当前状态来源节点
- **那么** 用户必须能够通过真实交互入口进入该历史节点详情或对应时间线节点
- **并且** 该入口必须引用与时间线详情相同的稳定节点标识
