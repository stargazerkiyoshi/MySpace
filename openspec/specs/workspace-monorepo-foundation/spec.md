## ADDED Requirements

### 需求:Monorepo 工作区结构
系统必须提供基于 `pnpm workspace` 的 monorepo 目录结构，以统一管理前端应用、主业务后端、AI 服务及根级共享工程配置。

#### 场景:初始化工程骨架
- **当** 开发者查看仓库根目录
- **那么** 仓库必须存在清晰的 workspace 配置以及用于承载多个应用的目录结构

### 需求:多应用统一脚本入口
系统必须在根目录提供统一的安装、开发、构建或启动脚本约定，避免每个应用使用完全不同的本地执行方式。

#### 场景:使用统一命令启动开发环境
- **当** 开发者按照项目文档执行根级命令
- **那么** 开发者必须能够定位并启动 `web`、`core-api`、`ai-service` 的开发流程

### 需求:共享工程配置基线
系统必须在 monorepo 根目录提供 TypeScript、环境变量和工程约定的共享基线，以避免三个应用之间出现无约束漂移。

#### 场景:新应用遵循统一约定
- **当** 开发者为现有应用增加配置或扩展新应用
- **那么** 新增配置必须可以继承或复用根级工程约定
