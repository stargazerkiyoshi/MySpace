## 新增需求

### 需求:Space 详情页必须承载 Node Graph View
系统必须在 Space 详情页中承载 Node Graph View，使用户除了节点列表和历史列表之外，还能在同一 Space 上下文中查看结构可视化视角。

#### 场景:进入某个 Space 后切换到 Graph View
- **当** 用户进入某个具体 Space 的详情页
- **那么** 页面必须提供进入 Node Graph View 的真实入口
- **并且** 用户必须能够在不离开当前 Space 上下文的前提下切换到该视图

## 修改需求

### 需求:Space 详情页必须承载 Timeline 区域
系统必须在 Space 详情页或 Space 首页中承载该 Space 的 Timeline 区域，以便用户看到最近发生的关键变化；当首页状态与历史形成联动后，该页面还必须承载当前状态与历史来源之间的最小联动入口；当 Space 开始支持结构可视化后，该页面还必须与 Node Graph View 一起构成同一 Space 工作区内的多视角表达。

#### 场景:进入某个 Space 后查看结构化 Timeline
- **当** 用户进入某个具体 Space 的详情页
- **那么** 页面必须展示该 Space 的 Timeline 列表区域
- **并且** 该区域必须能够让用户感知节点是否属于主干或分支

#### 场景:从当前状态进入相关历史节点
- **当** 用户在 Space 首页或当前状态区域查看某个历史来源摘要
- **那么** 页面必须提供进入相关历史节点的真实入口
- **并且** 用户进入后必须能继续在同一 Space 上下文中理解该节点与当前状态的关系
