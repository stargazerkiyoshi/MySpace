## MODIFIED Requirements

### 需求:占位路由页面
系统必须为核心页面提供路由入口，包括但不限于空间、节点、时间线、仪表盘等后续扩展入口；其中 Space 路由必须从占位入口演进为最小可用业务闭环，支持列表展示、创建入口和详情进入。

#### 场景:切换到 Space 页面
- **当** 用户访问 Space 路由
- **那么** 系统必须展示 Space 列表或空状态，而不是仅显示静态占位内容
- **并且** 页面中必须存在创建新 Space 的最小交互入口

#### 场景:进入单个 Space
- **当** 用户在 Space 列表中选择某个 Space
- **那么** 系统必须跳转到对应 Space 的详情页或首页占位页
- **并且** 详情页必须展示该 Space 的最小基本信息
