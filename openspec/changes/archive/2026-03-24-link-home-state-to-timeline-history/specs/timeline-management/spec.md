## 新增需求

### 需求:时间线节点必须表达其与当前状态的关系
系统必须让时间线节点详情能够表达该节点是否仍影响当前状态、是否进入当前主线，以及其影响类型。

#### 场景:查看时间线节点与当前状态关系
- **当** 用户打开某个时间线节点详情
- **那么** 系统必须返回该节点与当前状态关系的最小结构字段
- **并且** 该结构必须能够表达是否仍影响当前状态与影响类型

## 修改需求

### 需求:查看时间线节点详情
系统必须让用户能够在某个 Space 中查看时间线节点详情，并通过详情理解该节点的含义、结构关系以及它与当前状态的关联。

#### 场景:打开时间线节点详情
- **当** 用户在某个 Space 中打开某个时间线节点的详情
- **那么** 系统必须返回该节点的标题、时间、类型、简述或描述、对当前状态的影响、前置节点、后继节点以及是否属于主干
- **并且** 系统必须返回该节点是否进入当前主线、是否仍影响当前状态以及影响类型等关系字段
- **并且** 节点详情必须足以支持用户理解该节点与当前状态、前后节点和主线关系
