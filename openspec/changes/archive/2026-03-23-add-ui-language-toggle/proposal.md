## 为什么

当前前端已经形成基础工程壳层和 Space 最小业务闭环，但 UI 文案仍然是单语散落实现。中英文切换会影响应用壳层、导航、页面文案、提示信息和后续所有新增功能，因此需要先建立稳定的前端双语基础，再继续扩展业务能力。

## 变更内容

- 为 `web` 增加前端 UI 文案的中英文切换能力，支持 `zh-CN` 与 `en` 两种语言。
- 提供一个轻量的前端语言状态方案，支持默认语言选择和 `localStorage` 持久化。
- 在当前已有的壳层、导航、Space 页面和公共提示中接入双语文案。
- 明确当前只翻译前端 UI 文案，不翻译数据库内容、用户输入和后端业务数据。
- 保持现有前端架构分层，不引入过重的 i18n 复杂度。

## 功能 (Capabilities)

### 新增功能
- `web-ui-localization`: 定义前端 UI 文案的中英文切换、状态持久化和文案组织方式。

### 修改功能
- `web-app-shell`: 在前端壳层规范中补充语言切换入口、公共文案与页面双语能力要求。

## 影响

- `apps/web` 中的应用壳层、导航、Space 页面、公共提示和共享文案组织
- 前端全局状态与浏览器持久化
- `openspec/specs/` 下与前端 UI 行为相关的规范
