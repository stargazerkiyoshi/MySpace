## 为什么

当前时间线已经能展示“最近发生了什么”，但用户仍然无法真正理解时间线节点的含义、结构关系以及它对当前空间状态的影响。若时间线只停留在平铺展示层，历史就仍然是“看见存在”，而不是“可追踪、可理解、可回看”的过程系统。

现在推进时间线节点详情与主干/分支基础结构，是为了让时间线从纯展示升级为最小可用历史系统，在不提前实现完整历史树和复杂分支逻辑的前提下，先把“看得懂、点得开、能分清主线与支线”的能力建立起来。

## 变更内容

- 为时间线节点补充详情展示能力，支持查看标题、时间、类型、简述、对当前状态的影响、前置节点、后继节点以及是否属于主干。
- 扩展时间线节点的数据模型，引入 `isMainline`、`parentNodeId`、`branchFromNodeId`、`mergeToNodeId` 和 `nodeType` 等基础结构字段。
- 增强时间线列表展示，使主干节点与分支节点在 UI 上可区分，关键节点获得更强强调。
- 在节点详情中补充与当前状态、前后节点和主线关系的表达，为后续 merge、回归主线和当前状态联动预留结构。
- 明确本次不实现完整历史树、复杂分支合并逻辑、自动生成 Agent 流程、外部系统同步写入和重型可视化交互。

### 修改功能
- `timeline-management`: 将时间线从最小事件列表升级为带节点详情和主干/分支基础结构的历史系统。
- `web-app-shell`: 在 Space 页面内补充时间线节点详情交互和主干/分支可感知展示要求。
- `space-management`: 在 Space 详情 / 首页中补充时间线节点结构关系与当前状态影响的承载要求。
