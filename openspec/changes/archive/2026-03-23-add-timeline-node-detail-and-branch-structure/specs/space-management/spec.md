## 修改需求

### 需求:Space 详情页必须承载 Timeline 区域
系统必须在 Space 详情页 / Space 首页中承载该 Space 的 Timeline 区域，以便用户看到最近发生的关键变化；当时间线具备基础结构后，该区域还必须支持用户理解主线 / 分支关系以及节点对当前状态的影响。

#### 场景:进入某个 Space 后查看结构化 Timeline
- **当** 用户进入某个具体 Space 的详情页
- **那么** 页面必须展示该 Space 的 Timeline 列表区域
- **并且** 该区域必须能够让用户感知节点是否属于主干或分支

#### 场景:在节点详情中理解当前状态关系
- **当** 用户打开某个时间线节点详情
- **那么** 页面必须能够表达该节点如何影响当前进展、它是否通向当前主线以及它与前后节点的关系
- **并且** 当前实现的结构和展示不得阻碍后续接入更完整的首页状态联动
