## 修改需求

### 需求:TimelineEvent 作为 Space 内的最小历史记录对象被建模
系统必须将 TimelineEvent 建模为归属于某个 Space 的独立主对象，并为其提供可持久化的最小字段集合，以支持关键变化记录、列表展示、节点详情查看以及主干 / 分支基础结构表达。

#### 场景:时间线节点具备基础结构字段
- **当** 系统存储某个时间线节点
- **那么** 该节点必须能够表达 `isMainline`、`parentNodeId`、`branchFromNodeId`、`mergeToNodeId` 和 `nodeType` 这些基础结构语义
- **并且** 这些结构信息必须进入稳定数据模型，而不是只存在于 UI 推导中

#### 场景:查看时间线节点详情
- **当** 用户在某个 Space 中打开某个时间线节点的详情
- **那么** 系统必须能够返回该节点的标题、时间、类型、简述 / 描述、对当前状态的影响、前置节点、后继节点以及是否属于主干
- **并且** 节点详情必须足以支持用户理解该节点与当前状态、前后节点和主线关系

### 需求:系统可以按 Space 维度读取 Timeline 列表
系统必须提供按某个 Space 维度读取 TimelineEvent 列表的能力，以便前端展示最近发生的关键变化，并区分主干节点与分支节点。

#### 场景:读取某个 Space 的结构化 Timeline 列表
- **当** 前端基于某个 Space 标识请求 Timeline 列表
- **那么** 系统必须只返回属于该 Space 的 Timeline 节点集合
- **并且** 返回结果必须包含用于区分主干 / 分支、关键节点和详情展示的最小结构字段

### 需求:Timeline 节点类型在当前阶段必须保持受控
系统必须将时间线节点类型收敛为受控集合，以支撑主线推进、分支产生、决策、外部事件、完成和中断的最小语义表达。

#### 场景:记录受支持的时间线节点类型
- **当** 系统写入或返回时间线节点类型
- **那么** 节点类型必须限制在主线推进、分支产生、决策、外部事件、完成或中断的受控集合
- **并且** UI 必须能够对这些类型提供中文文案，并兼容现有双语体系
