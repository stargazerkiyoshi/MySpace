## 修改需求

### 需求:Space 页面必须展示 Timeline 列表区域
系统必须在现有 Space 页面壳层能力上补充 Timeline 列表区域，以便用户在同一页面内同时看到内容单元和最近变化；当时间线升级为结构化历史系统后，页面还必须支持时间线节点详情查看以及主干 / 分支的可感知区分。

#### 场景:进入某个 Space 后查看结构化时间线
- **当** 用户从 Space 列表进入某个具体 Space
- **那么** Space 详情页必须展示该 Space 下最近的 TimelineEvent 列表
- **并且** 主干节点与分支节点在视觉上必须可以区分
- **并且** 当前节点或关键节点必须可以获得更高强调

#### 场景:在 Space 页面内查看时间线节点详情
- **当** 用户点击某个时间线节点
- **那么** 页面必须展示该节点的详情区域、面板或等价交互
- **并且** 用户必须能够在当前 Space 上下文中理解该节点的关系与影响，而不是只能看到平铺列表项
