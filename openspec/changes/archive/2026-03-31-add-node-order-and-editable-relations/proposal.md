## 为什么

当前 Space 画布里的节点前后顺序只能靠创建时间和自动布局猜测，用户很难直接判断“哪个节点应该在前”。同时，连线完全由 Timeline 派生，用户无法删除错误连线，也无法按自己的理解重新连接节点，导致图视图只能看，不能整理。

现在推进这项变更，是为了让节点画布具备最小可控性：用户可以明确节点顺序，并对连线做删除和重连，从而让图视图真正成为空间结构表达的一部分，而不是只读投影。

## 变更内容

- 为 Node 引入稳定的显示顺序字段，并在创建节点时自动追加到末尾。
- 在 Space 的图视图中按节点顺序稳定排布，避免新节点跳到已有节点左侧。
- 为图视图引入可持久化的节点关系模型，允许用户删除已有连线并手动新建连线。
- 保留 Timeline 作为历史记录来源，但不再把它当作图视图唯一的关系真相源。

## 功能 (Capabilities)

### 新增功能

- `node-relations`: 管理 Space 内节点之间可编辑、可持久化的结构关系。

### 修改功能

- `node-management`: Node 需要具备稳定顺序，并在创建后可参与顺序化展示。
- `space-node-visualization`: Graph View 需要按稳定顺序排布节点，并支持删除、重连连线。

## 影响

- `apps/core-api` 的 Node / Graph / Timeline 相关应用服务、仓储和 Prisma 数据模型。
- `apps/web` 的 Node Graph View、节点创建后刷新行为，以及图上的删除/连线交互。
- `apps/core-api/prisma/schema.prisma` 和对应迁移。
