## ADDED Requirements

### 需求:Graph View 必须支持显式节点关系编辑
系统必须允许用户在 Space 的 Graph View 中删除已有连线并手动新建连线，以便图结构可以被用户直接整理，而不是只能依赖自动推导。

#### 场景:删除已有连线
- **当** 用户在 Graph View 中删除一条已存在的节点连线
- **那么** 系统必须持久化删除该关系
- **并且** 该连线在刷新后不得再次自动出现，除非用户重新创建它

#### 场景:手动新建连线
- **当** 用户在 Graph View 中从一个节点连接到另一个节点
- **那么** 系统必须持久化该节点关系
- **并且** 刷新后必须仍然返回这条连线

### 需求:Graph View 必须基于节点稳定顺序布局
系统必须在 Graph View 中按节点稳定顺序布局，以便用户能够直接判断哪个节点在前、哪个节点在后。

#### 场景:读取图数据时按顺序布局
- **当** 前端基于某个 Space 请求 Graph View 数据
- **那么** 系统必须按节点稳定顺序返回布局位置
- **并且** 新创建节点必须默认出现在当前顺序的末尾区域，而不是插入到已有节点左侧

## MODIFIED Requirements

### 需求:Space 必须提供基于真实数据的 Node Graph View
系统必须允许用户在某个 Space 中查看基于真实 Node 数据构建的 Graph View，以便直接观察节点结构；该视图不得依赖脱离真实数据的演示图。

#### 场景:读取某个 Space 的 Graph 数据
- **当** 前端基于某个 Space 标识请求 Node Graph View 数据
- **那么** 系统必须返回该 Space 的 Node 集合与对应的 Edge 集合
- **并且** Node 必须映射自真实的 Space / Node 数据
- **并且** Edge 必须映射自系统持久化的节点关系数据
- **并且** 当当前 Space 不存在已保存的节点关系时，系统必须返回空 edge 集合，而不是伪造演示连线

### 需求:Graph View 必须支持最小画布交互
系统必须在 Node Graph View 中支持缩放、拖拽和平移这些基础画布交互，使用户能够浏览不同密度下的节点结构。

#### 场景:在 Graph View 中浏览画布
- **当** 用户进入某个 Space 的 Node Graph View
- **那么** 用户必须能够缩放、拖拽节点和移动画布
- **并且** 这些交互不得要求用户切换到独立编辑模式

### 需求:选中节点必须驱动画内信息联动
系统必须在 Graph View 中支持节点选中态，并让选中结果与同页信息面板或详情区联动，以便用户在图中定位节点后继续理解其内容与关系。

#### 场景:在 Graph View 中选中节点
- **当** 用户在 Graph View 中点击某个 Node
- **那么** 系统必须将该 Node 标记为当前选中节点
- **并且** 页面必须同步展示该 Node 的基础信息或详情摘要

### 需求:Graph View 必须支持最小邻接聚焦
当 Graph 数据中存在边关系时，系统必须能够围绕当前选中节点突出显示其直接关联节点，以帮助用户理解局部结构。

#### 场景:聚焦选中节点及其直接关联节点
- **当** 用户在存在边关系的 Graph View 中选中某个 Node
- **那么** 系统必须能够突出显示该 Node 与其直接关联节点
- **并且** 系统必须允许用户在当前画布上下文中完成局部聚焦，而不强制跳转离开 Graph View
