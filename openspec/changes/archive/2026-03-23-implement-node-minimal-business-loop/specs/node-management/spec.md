## ADDED Requirements

### 需求:Node 作为 Space 内的独立内容单元被建模
系统必须将 Node 建模为归属于某个 Space 的独立主对象，并为其提供可持久化的最小字段集合，以支持创建、列表展示、详情查看和基础更新。

#### 场景:创建 Node 后持久化主对象
- **当** 用户在某个 Space 下发起创建 Node 请求
- **那么** 系统必须将新 Node 记录写入数据库
- **并且** 该记录必须至少包含 `id`、`spaceId`、`title`、`content`、`nodeType`、`status`、`createdAt` 和 `updatedAt`

### 需求:用户可以在某个 Space 下创建 Node
系统必须允许用户在某个 Space 下创建新的 Node，且当前阶段不得要求复杂权限或关系建模流程。

#### 场景:成功创建 Node
- **当** 用户提交合法的 Node 创建信息且目标 Space 存在
- **那么** 系统必须创建新的 Node
- **并且** 返回结果必须包含可供前端刷新列表和进入详情所需的基础字段

### 需求:用户可以查看某个 Space 下的 Node 列表
系统必须提供按 Space 维度读取 Node 列表的能力，以便前端在某个 Space 内展示当前内容单元集合。

#### 场景:读取某个 Space 下的 Node 列表
- **当** 前端基于某个 Space 标识请求 Node 列表
- **那么** 系统必须只返回属于该 Space 的 Node 集合
- **并且** 列表结果必须包含渲染列表项和进入详情所需的最小字段

### 需求:用户可以查看 Node 详情
系统必须提供 Node 详情读取能力，以便前端查看某个具体 Node 的完整最小内容。

#### 场景:读取单个 Node 详情
- **当** 前端基于 Node 标识请求详情
- **那么** 系统必须返回该 Node 的完整最小字段集合
- **并且** 返回结果必须足以支撑前端展示与基础编辑

### 需求:用户可以更新 Node 基础字段
系统必须允许用户对 Node 的基础字段进行更新，包括标题、内容、类型和状态，且不得要求复杂版本管理或协作流程。

#### 场景:成功更新 Node
- **当** 用户提交合法的 Node 更新信息
- **那么** 系统必须更新目标 Node 的基础字段
- **并且** 返回更新后的 Node 基础信息
