## 新增需求

### 需求:Space 工作区必须提供次级信息的显式展开入口
系统必须在 Space 工作区中为 Space 详情、节点详情和最近变化提供显式展开入口，以便这些信息在默认隐藏后仍然可访问。

#### 场景:按需展开次级信息
- **当** 用户位于某个 Space 的工作区页面
- **那么** 页面必须提供查看 Space 详情、节点详情或最近变化的真实触发入口
- **并且** 用户必须能够在不离开当前 Space 上下文的前提下完成展开与关闭

## 修改需求

### 需求:Space 工作区必须支持结构视图切换入口
系统必须在 Space 工作区中提供进入 Node Graph View 的切换入口，使用户能够在列表、历史和结构可视化之间切换，而不离开当前 Space 页面；但默认进入 Space 页面时，结构视图不得替代当前进展节点成为首屏主表达。

#### 场景:在 Space 工作区切换到 Graph View
- **当** 用户位于某个 Space 的工作区页面
- **那么** 页面必须提供进入 Node Graph View 的切换入口
- **并且** 该入口必须与现有工作区视图保持一致的页面承载方式

### 需求:Graph View 必须承载图画布与信息面板的最小布局
系统必须在 Graph View 中承载图画布与信息面板的最小布局，以便用户在浏览结构时能够同时看到选中节点的上下文信息；但该布局默认不得占据 Space 页面首屏。

#### 场景:在 Graph View 中查看选中节点信息
- **当** 用户在 Graph View 中选中某个 Node
- **那么** 页面必须在同一工作区内展示与该节点相关的基础信息面板或详情摘要
- **并且** 页面不得强制跳转到独立详情路由才能完成这一步
