## MODIFIED Requirements

### 需求:用户可以查看 Space 详情
系统必须提供 Space 详情读取能力，以便前端进入某个具体 Space 并展示其首页承载内容；该详情页不仅必须展示 Space 的最小基础信息，还必须作为 Space 内 Node 内容单元的第一版承载入口。

#### 场景:读取单个 Space 详情
- **当** 前端基于 Space 标识请求详情
- **那么** 系统必须返回该 Space 的基础信息
- **并且** 返回结果必须足以支撑 Space 首页占位页渲染与 Node 区域承载

### 需求:前端必须形成 Space 最小闭环
系统必须在前端提供 Space 列表页、最小创建入口和 Space 详情页，使用户能够完成创建、查看和进入 Space 的闭环；当用户进入某个 Space 后，详情页还必须承载该 Space 下 Node 内容单元的最小闭环入口。

#### 场景:用户从列表创建并进入 Space
- **当** 用户在前端 Space 列表页创建一个新的 Space
- **那么** 前端必须调用后端创建接口并展示创建结果
- **并且** 用户必须能够在列表中看到该 Space
- **并且** 用户必须能够点击进入该 Space 的详情页或首页占位页

#### 场景:用户在 Space 内继续处理 Node
- **当** 用户已经进入某个 Space 的详情页
- **那么** 页面必须展示该 Space 下的 Node 列表和当前选中 Node 的基础信息
- **并且** 用户必须能够在该页面内完成 Node 的创建、查看与基础修改
